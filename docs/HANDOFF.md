## Last session's bug-fix sequence

The user playtested four times and reported five categories of failure. Each was addressed:

1. **Particle Velocity curves spam** → fixed in `ParticleBuilder.cs` by setting x, y, z components with matching `MinMaxCurve(min, max)` ctors
2. **No menu text** → `Assets/Blech/Editor/TmpEssentialsImporter.cs` uses `[InitializeOnLoad]` to auto-import the TMP unitypackage from `Library/PackageCache/.../Package Resources/`
3. **"5 hot dogs and 2 eyeballs"** then **"floating eyes only"** → after two attempts at Quaternius FBX loading (which kept being unreliable), gave up and made the body a stretched sphere primitive in `PlayerPrefabBuilder.cs`
4. **Random beeping** → was the WindHazard + AcidGeyser firing 2D SFX across the entire level every ~4-7s from biomes the player wasn't in. Fixed by gating WindHazard SFX on `_players.Count > 0` and removing AcidGeyser SFX entirely. Then added actual music via `MusicPlayer.cs` + `MusicPrefabBuilder.cs`.
5. **No jump** → first attempt was rewriting `PlayerInput.cs` to define InputActions in code instead of loading the `.inputactions` asset. That didn't fix it. Second attempt (current HEAD) replaced InputActions entirely with direct `Keyboard.current` polling, plus added 0.2s coyote time in `PlayerMovementController.OnJump`. **User has not re-tested this yet.**

## Useful diagnostic snippets

Force Unity to re-import everything if Editor cache is stale:
```bash
# Close Unity Editor first
rm -rf Library/ScriptAssemblies Library/Bee
./tools/build-all.sh
```

Check what music track is being used:
```bash
grep -A1 "Music track" /dev/stderr <<< "$(./tools/build-all.sh 2>&1)" | tail -3
```

Find Kenney audio matching a keyword:
```bash
find "Assets/Blech/_ThirdParty/Kenney Game Assets All-in-1 3.4.0/Audio" -iname "*jump*"
```

## Conventions

- Don't add files to scenes via the Inspector. Every prefab/material/shader/scene is generated.
- If you need to add a new asset type, write a `XxxBuilder.cs` under `Assets/Blech/Editor/`, hook it into `BlechEditorMenu`, and call it from `BuildAll()`.
- Edit existing files; don't proliferate new ones unless there's a real boundary.
- Commits are small and focused. Prefer many small commits over one big one.
- No git operations the user didn't ask for (no `--force`, no force-push to main without explicit asking).
